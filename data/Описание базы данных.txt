База данных состоит из двух таблиц:
1 - waters - таблица, содержащая информацию о водоемах:
1.1 - water_id - идентификатор водоема, автоматически генерируемое значение. Столбец не может содержать значение NULL и является первичным ключем.
1.2 - product_name - наименование водоема, столбец не может содержать значения NULL.
1.3 - category - ссылка на категорию водоема в таблице "categories". Столбец не может содержать значение NULL. Автоматически категория задается как "море".
1.4 - date - дата замера всех показателей. Столбец не может содержать NULL. Дата записывается в виде ДД.ММ.ГГГГ.
1.5 - water_level - уровень воды относительно точки измерения "нуля поста". Столбец может содержать NULL, например, данные за этот день еще не поступили.

2 - categories - таблица будет содержать описание категорий водоемов:
2.1 - category_id - идентификатор категории, данное значение должно автоматически генерироваться. Столбец не может содержать значение NULL и является первичным ключом.
2.2 - сategory_name - наименование категории, столбец не может содержать значение NULL.
2.3 - water_level_limit - нормированное значение уровня воды, при преодолении которого водоем считается опасным (происходит выход из берегов и затопление прибрежных территорий). Столбец не может содержать значение NULL. Автоматически задается 1.
